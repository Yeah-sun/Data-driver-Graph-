# 岗位能力图谱 UI 原型文档

## 1. 设计方向

本原型是工程级产品工作台，不是营销页面。界面参考 GitHub Primer 的产品语言：清晰边框、紧凑列表、状态标签、可扫描的操作行、浅色工程界面。同时加入岗位能力图谱自己的识别点：图谱网格、节点关系、任务轨迹、版本演化。

## 2. 工程结构

- 入口：`index.html`
- 框架：Vue 3
- 构建：Vite
- 路由：Vue Router Hash History
- 基础 UI：`@primer/css`
- 图标：`@primer/octicons`
- Mock 数据：`src/data/mock.js`

## 3. 页面说明

### 登录 / 注册

功能目标：为系统补齐账户入口，并按角色分流。学生、教师和游客进入业务工作台；管理员进入管理后台，不参与简历、岗位匹配和学习路径等个人业务流程。

用户输入：
- 学校 / 单位
- 身份角色
- 账号
- 密码
- 注册时的姓名和确认密码

页面输出：
- 前端 mock 会话
- 学生、教师和游客登录后跳转到业务工作台
- 管理员登录后跳转到管理后台
- 未登录访问业务页或后台页时自动回到登录页
- 管理员不能访问个人业务页，普通用户不能访问管理后台

主要按钮：
- `登录`：校验基础字段，并根据身份进入对应入口
- `注册并进入`：创建前端 mock 账户，并根据身份进入对应入口
- `游客演示`：不填写账户，直接进入演示身份

说明：当前账户只存在浏览器 `localStorage` 中，不提交服务器。后续接真实后端时替换为 Auth API 或学校统一身份认证。

### 我的工作台

功能目标：展示用户当前进度、推荐动作、关键指标和后台任务。

用户输入：
- 点击快速动作
- 点击继续主流程

页面输出：
- 当前流程轨迹
- 关键指标
- 推荐下一步
- 异步任务状态

主要按钮：
- `继续主流程`：跳转到简历中心
- `确认简历画像`：跳转到简历中心
- `比较两个目标岗位`：跳转到岗位探索
- `打开岗位能力图谱`：跳转到能力图谱

### 简历中心

功能目标：上传简历、查看解析步骤、展示技能画像和待复核项。

用户输入：
- 简历文件
- 技能复核操作

页面输出：
- 文件列表
- 解析步骤状态
- 技能标签和熟练度
- 待复核技能列表

主要按钮：
- `选择文件`：模拟上传入口
- `编辑技能`：预留技能编辑
- `保存并选岗位`：跳转到岗位探索

### 岗位探索

功能目标：筛选岗位、浏览岗位列表、查看岗位详情并设为目标。

用户输入：
- 筛选条件
- 岗位选择

页面输出：
- 岗位列表
- 岗位热度与匹配度
- 必备技能与加分技能

主要按钮：
- 筛选按钮：切换筛选条件
- `加入对比`：预留岗位对比
- `设为目标`：跳转到匹配分析

### 匹配分析

功能目标：展示综合匹配分、分项解释、命中技能和缺口。

用户输入：
- 当前简历画像
- 目标岗位

页面输出：
- 综合评分
- 分项评分
- 命中技能
- 主要缺口
- 系统结论

主要按钮：
- `查看差距路径`：跳转到学习路径

### 学习路径

功能目标：将技能差距转化成可执行路线。

用户输入：
- 匹配结果
- 路线生成操作

页面输出：
- 差距 backlog
- 六周路线
- 导出入口

主要按钮：
- `重新生成`：预留重新生成路线
- `导出方案`：预留文档导出

### 能力图谱

功能目标：展示岗位与技能的关系网络，以及岗位能力演化。前端按 Neo4j 图数据结构渲染，支持节点拖拽、缩放、力导向重排、点击节点查看属性和展开邻居节点。

用户输入：
- Neo4j 图谱数据
- 节点点击
- 节点拖拽
- 缩放和力导向重排
- 展开子技能或邻居节点

页面输出：
- 可交互关系画布
- 节点说明和直接关系
- 版本演化时间线

主要按钮：
- `刷新图谱`：请求 `/api/graph/neo4j?job=...` 获取 Neo4j 图谱数据
- `力导向重排`：前端重新计算节点布局
- `展开子技能`：预留 Neo4j 邻居节点展开

### 管理后台

功能目标：保留比赛闭环需要的用户管理、数据导入、岗位维护和测试评估。

用户输入：
- 用户账号、角色和状态处理动作
- 数据源文件
- 新岗位定义动作
- 既有岗位能力更新动作
- 评测执行动作

页面输出：
- 用户列表、角色标签、账号状态和最近活跃时间
- 数据导入任务
- 新岗位发现与定义卡片
- 既有岗位能力更新卡片
- 测试中心指标

主要按钮：
- `新建用户`：预留管理员创建账号入口
- `批量导入账号`：预留学校/班级账号导入
- `重置密码`：预留账号安全处理
- `审核通过`：预留教师或管理员账号审核
- `禁用账号`：预留异常账号限制
- `选择数据源`：预留数据导入
- `定义新岗位`：预留管理员确认岗位名称、职责边界和核心能力
- `保存更新草稿`：预留既有岗位能力变化暂存
- `发布到图谱`：预留同步图谱
- `一键执行评测`：预留测试任务

## 4. 后续接口预留

本阶段不定义详细字段，只保留接口方向：

- Resume API：上传、解析、复核、保存画像
- Job API：筛选、岗位详情、岗位对比
- Matching API：发起匹配、获取报告
- Learning API：生成路线、导出方案
- Graph API：获取 Neo4j 图谱、展开节点、查看演化
- Admin API：管理用户、导入数据、维护岗位、执行评测

## 5. 验收重点

- 7 个页面都能通过左侧导航访问
- 页面风格统一，参考 Primer 但不逐像素复刻 GitHub
- 不使用紫色 AI 风或营销页 hero
- 页面在窄屏下能纵向折叠
- `npm run build` 能通过
